hexo博客网站搭建流程

发表于2021-11-20|更新于2021-11-20|建站
|字数总计:362|阅读时长:1分钟|阅读量:
  • 平台: Windows
  • 博客框架: hexo
  • 部署方式: github pages

第一部分

初步搭建hexo框架, 部署到Github Pages, 绑定个人域名

1. 安装nodejs和npm工具

https://nodejs.org/en/download/
windows下安装nodejs将同时安装包管理工具npm.

2. 安装hexo并初始化

安装hexo: npm install hexo-cli
初始化hexo框架: hexo init [blog_dir]
完成初始化后, 目录下将生成若干子目录和文件, 其中包括:

  • node_modules: 依赖包
  • public: 存放生成的页面
  • scafflods: 用于生成博文的模板
  • source: 存放你写的文章
  • themes: 博客主题
  • _config.yml: 主配置文件

常用hexo命令:

  • 生成博文: hexo generate或hexo g
  • 本地部署: hexo server或hexo s
  • 远程部署: hexo deploy或hexo d
  • 清空生成目录: hexo clean或hexo c

3. 部署到Github Pages

在Github创建名称为[github_name].github.io的新仓库.
修改站点配置文件_config.yml的deploy字段:

安装hexo的git部署工具: npm install hexo-deployer-git --save
完成上述步骤后执行hexo d进行部署. 看到以下输出表示已部署成功.

4. 配置个人域名(可选)

成功部署到Github Pages后, 默认域名是.github.io. 要想使用个人域名(可从云服务供应商购买域名, 如阿里云), 需前往Github仓库的settings下的pages选项卡下修改.
